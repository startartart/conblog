### 프로젝트 : 하나의 프로그램(시스템)을 만들기 위한 일련의 프로세스를 뜻한다.

- 어떤 프로그램을 만들기 위해 `기획 - 설계 - 테스트 - 배포` 단계를 거친다.
- 프로젝트는 개인, 팀으로 다양하게 진행할 수 있다.
- 팀 프로젝트의 경우, 팀원간 협업이 필수이며 개발자의 경우 커뮤니케이션 즉 소통이 원활하게 이루어 져야한다.

### Readme : 완성된 프로그램의 설명서, 구현중인 프로젝트의 현황 등을 확인할 수 있다.

- 하나의 문서로도 사용할 수 있으며, 프로젝트 진행간 협업을 원활하게 진행시켜주는 수단 중 하나이다.
- 다양한 확장자를 사용할 수 있으나, 대표적으로 `markdown(.md)`를 많이 채용한다.

### Markdown : 텍스트 기반의 마크업 언어로, 문서 작성에 사용된다.

- 일반 사용자에게 쉬운 조작법과 스타일을 제공한다. 텍스트를 웹 기술로 바꿔주는 하나의 도구로 볼 수 있다.
- 다양하게 변형되어 사용될 수 있으며, 대표적으로 **Github**에서 사용되는 **Github-Flavored Markdown**이 존재한다.

### 버전 관리(VCS : Version Control System)

- 버전이란 유의미한 수정을 의미한다.
- 버전 관리는 크게 **로컬 VCS, 중앙집중식 VCS, 분산 VCS**로 분류할 수 있다.

  1. **로컬 VCS** : 내 컴퓨터에서만 관리하므로 협업이 어렵다.
  2. **중앙집중식 VCS** : 중앙 서버로부터 필요 파일만 가져와 작업하므로 협업이 가능하다.

     `ex) SVN, CVS`

  3. **분산 VCS** : 중앙 서버로부터 모든 파일을 가져와 작업하므로 중앙집중식 VCS에 비해 안정적으로 관리할 수 있다.

     `ex) Git, Mecurial, Bazaar`
